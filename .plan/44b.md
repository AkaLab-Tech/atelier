# Plan — #44b Rewire /atelier:align Tier-3 auto to delegate the whole coordination to the orchestrator

- **Status:** ready (approved — product lead, batch approval via AskUserQuestion, 2026-07-07)
- **Type / priority / estimate:** bug · P1 · ~TODO (suggest ~1.5h — one command section rewrite + two test updates)
- **blocked_by:** #44a (the orchestrator non-task mode + corrected invariant must exist first)

## Approach

`commands/align.md` Tier-3 under `auto` (lines ~110–139) is the main offending site: it dispatches `pr-opener` to author the base PR, then — **from the align session itself** — invokes `reviewer` (via `SlashCommand`) and `/atelier:auto-merge` (steps 1–2 of that section). That collapses the authoring actor and the reviewing actor into the align session, which is exactly the self-approval trip #44 fixes. Rewire it to hand the **whole** coordination one level down to the generalized `task-orchestrator` (non-task mode from #44a):

1. **Keep base-PR preparation in the align session.** The temp-worktree setup is unchanged: resolve the base branch, `git -C <repo> worktree add -b chore/atelier-align <tmp> origin/<base>`, copy in the changed `.atelier.json` / settings / adopted tracking, and commit (Conventional Commits, no AI attribution). Preparing a worktree is not authoring a PR — it does not push or `gh pr create`, so it does not implicate the classifier.

2. **Replace the pr-opener-then-self-review block with a single orchestrator dispatch per repo.** Instead of `Task(pr-opener)` followed by the align session's own `reviewer` + `auto-merge` steps, dispatch `task-orchestrator` (via `Task`) in non-task mode with `{ mode: non-task-pr, repo, worktree (the temp worktree), base, head: chore/atelier-align, title, body, interactive: false }`. The orchestrator dispatches `pr-opener` (author) → `reviewer` → `auto-merge` as *its* sub-agents; the align session never itself dispatches `reviewer` or `auto-merge`. This is the property that clears the classifier: the align session is not the coordinating actor for author+review — the orchestrator, one level down, is.

3. **Move the post-merge local fast-forward to after the orchestrator returns `merged`.** Step 3 of the current auto section (`git -C <repo> pull --ff-only` on the operator's checkout) stays in the align session — it is a local pull, not a PR coordination step — but now runs on the orchestrator's `merged` return. Remove the temp worktree once the orchestrator reports terminal (merged/held), as today.

4. **Handle the orchestrator's terminal states.** On `merged`: fast-forward + cleanup + report. On `held` (a §6 guardrail held, e.g. unprotected-repo empty `reviewDecision`): surface the held reason and stop — do **not** bypass, do **not** call `gh pr merge`, do **not** prompt the operator to merge manually (the existing hard rules already say this; keep them, just retarget them from the inline reviewer/auto-merge steps to the orchestrator's return).

5. **Leave the Tier-3 `ask` path completely unchanged.** Under `ask`, the operator merges on their own timeline; align still offers to open the PR inline via `AskUserQuestion` and never coordinates review. This path never trips the classifier (no self-review) and is a deliberate no-regression surface the existing test guards.

6. **`allowed-tools` frontmatter.** `Task` is already granted. `SlashCommand` was used to invoke `reviewer` inline — after the rewire the align session no longer invokes `reviewer`/`auto-merge` directly, so `SlashCommand` may become unused for the auto path; keep it only if still needed for Tier-2 delegations (`/adopt-roadmap`, `/onboard-workspace`) — it is, so leave the frontmatter as-is. `Bash(gh:*)` stays for survey/worktree ops.

## Acceptance criteria

- [ ] `commands/align.md` Tier-3 `auto` delegates the whole author→review→merge coordination to `task-orchestrator` (non-task mode) per repo; the align session prepares the worktree but never dispatches `reviewer` or `/atelier:auto-merge` itself.

- [ ] `hooks/tests/pr-opener-delegation.test.sh` and `hooks/tests/align-auto-policy.test.sh` pass against the rewired command, asserting the orchestrator-delegation contract and the preserved `ask`-path behaviour.

## Risks / open questions

- **Per-repo orchestrator dispatch cost.** A multi-member workspace now spawns one `task-orchestrator` sub-agent per changed repo (Opus) instead of an inline reviewer call. That is the correct trade for classifier-safety, but the implementer should confirm the align command already iterates per-repo (it does) so the dispatch nests cleanly in that loop.
- **Non-task-mode briefing completeness.** The align dispatch must pass every field the #44a non-task mode requires (`repo`, `worktree`, `base`, `head`, `title`, `body`, `interactive: false`). If #44a's guard field name changes during implementation, this dispatch must match it — a coordination point between the two sub-tasks the implementer should verify against the merged #44a.
- **`held` on unprotected member repos.** Per memory `auto-merge-needs-branch-protection`, member repos are unprotected, so guardrail #2 (`reviewDecision`) may hold even with a real approval. The rewire does not fix that (separate issue); the report must make "delegated, but held on branch-protection" legible so the operator isn't misled into thinking the delegation failed.
