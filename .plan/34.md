# Plan — #34 github-project mirror reconciliation must read `origin/<base>` and run on `SessionStart`

- **Status:** ready (approved — product lead)
- **Type / priority / estimate:** bug · P1 · ~3h
- **blocked_by:** none (but see *Risks* — sequencing vs. `#32`)

## Approach

The github-project backend keeps `ROADMAP.md` / `HISTORY.md` / `IN_PROGRESS.md`
as an **offline mirror** of the Project board (`.roadmap.json`:
`backend: github-project`, `offlineMirror: true`). The board is the source of
truth; the mirror is regenerated by crt's `backend → files` reverse engine
(PLAN.md §16.6, shipped by `#21a`/`#21b`, reached via `/migrate-roadmap --to files`
/ `listTasks` across the three buckets). The bug is that nothing in atelier-core
converges that mirror against the **merged truth** automatically — the only
write path is the (soft, per #32) closing-PR edit, and any reconcile that exists
reads the **local working checkout**, which lags `origin/<base>` by exactly the
un-pulled merge commits, and is gated by a **main-checkout path glob**
(`*/<repo>/ROADMAP.md`) that never matches atelier's own edits because those
happen inside worktrees (`<repo>-worktrees/<branch>/ROADMAP.md`). Net effect
(deminut `#28` evidence): a delivered task stays `Todo` on the mirror until the
operator manually `git pull` + re-runs a sync.

The fix has two structural corrections, mirroring the task's own scope sketch:

1. **Source = the merged truth, not the local checkout.** Reconcile against
   `origin/<base>` (`base` = `dev` if `origin/dev` exists, else `main` — the same
   probe `commands/next-task.md` step uses: `git ls-remote --heads origin dev`).
   `git fetch origin <base>` then read the merged mirror with
   `git show origin/<base>:ROADMAP.md` (and `HISTORY.md` / `IN_PROGRESS.md`) —
   never the local file, never a worktree-relative path. This is the exact
   discipline `/next-task` already follows for its backlog source
   (`commands/next-task.md` lines 32/51/198: "read it with
   `git show origin/<base>:ROADMAP.md` … The local working copy is **not** the
   source of truth"), so #34 brings the *mirror reconcile* in line with the
   already-correct *backlog read*.

2. **Trigger = `SessionStart`, not a path-filtered edit hook.** Add a fourth
   `SessionStart` hook alongside the existing three (`hooks/hooks.json`:
   `load-operator-rules.sh`, `daily-housekeeping.sh`, `orient-session.sh`),
   following the established hook→helper split (`orient-session.sh` →
   `scripts/atelier-orient`; `daily-housekeeping.sh` → `atelier-housekeeping`).
   The hook is a thin, fail-open dispatcher; the helper does the work and is
   **gated** to non-`files` backends with `offlineMirror: true` (resolve backend
   via `scripts/atelier-task-backend "$dir"`; read `.offlineMirror` with `jq`).
   For every other project (`files` backend, or `offlineMirror: false`) the
   helper is a silent no-op — no fetch, no output.

**Bash/AI split (the load-bearing design decision — see Risks).** A bash
`SessionStart` hook cannot drive the GitHub MCP / OAuth that crt's `listTasks`
needs, and the existing two cheap hooks deliberately touch *no* git/gh to avoid
latency, permission prompts, and premature OAuth. So split the reconcile:

- **Cheap bash half (this task's hook + helper):** `git fetch origin <base>`,
  then compare `git show origin/<base>:ROADMAP.md` against the local mirror. If
  the merged mirror is ahead (the common case once #32's closing-PR gate keeps
  `origin/<base>` mirror == board), **guarded fast-forward** the local base from
  `origin/<base>` — reusing the postMergeCleanup `fastForwardBase` guards
  (`.atelier.json`): dirty/diverged main worktree → **skip with a surfaced note,
  never force**. Print a one-line drift/converged summary to stdout (SessionStart
  stdout becomes session context, per the other hooks).
- **AI-driven half (board → mirror regen):** when the bash half detects the
  board may be ahead of `origin/<base>` itself (e.g. an operator set `Status=Done`
  on the board with no PR, or #32's gate has not landed), the hook **surfaces a
  one-line instruction** for the session to run the crt reverse engine
  (`/migrate-roadmap --to files`) and open the mirror-regen PR — atelier-core
  *drives* crt, it does not re-implement the reverse engine.

Keep the per-session cost bounded with a **once-per-day stamp** under
`$ATELIER_CONFIG_DIR` (exactly the `daily-housekeeping.sh` pattern: stamp file,
`[ "$LAST" = "$TODAY" ] && exit 0`) so the `git fetch` does not run on every
single session start. A new `commands/sync-mirror.md` (or reuse an existing
migrate command) gives the operator an explicit on-demand reconcile, matching
the housekeeping hook↔command split.

Steps, in order:
1. `scripts/atelier-mirror-reconcile` (new helper): resolve `$dir`, resolve
   backend + `offlineMirror`, no-op unless `non-files && offlineMirror`. Resolve
   `base`, `git fetch origin <base>` (fail-open on no network), diff
   `origin/<base>:ROADMAP.md` vs local mirror, guarded fast-forward when behind,
   print converged/drift/skip summary. `--help`, `set -uo pipefail`, exit 0
   always (fail-open to silence), `$CLAUDE_PLUGIN_ROOT`-relative — mirror the
   `atelier-orient` / `atelier-task-backend` conventions exactly.
2. `hooks/reconcile-mirror.sh` (new hook): cheap dispatcher with the daily stamp
   guard; resolve the helper (PATH then `$CLAUDE_PLUGIN_ROOT/scripts`); print its
   stdout. Mirror `orient-session.sh` structure line-for-line.
3. Register the hook as a fourth `SessionStart` entry in `hooks/hooks.json`.
4. Hermetic regression test in `hooks/tests/` (see Acceptance) — backend-gated,
   origin-sourced, worktree-path-independent, SessionStart-wired.
5. Document the convergent-reconcile design under PLAN.md §16 (new sub-section
   near §16.6, the offline-mirror / reverse-path home) — CLAUDE.md requires the
   plan and the change land together.

## Affected areas

- `hooks/hooks.json` — add a fourth `SessionStart` hook entry pointing at
  `"${CLAUDE_PLUGIN_ROOT}"/hooks/reconcile-mirror.sh`.
- `hooks/reconcile-mirror.sh` *(new)* — thin, fail-open `SessionStart` dispatcher
  with a once-per-day stamp; delegates to the helper and echoes its summary.
  Patterned on `hooks/orient-session.sh` + `hooks/daily-housekeeping.sh`.
- `scripts/atelier-mirror-reconcile` *(new)* — the reconcile helper: backend +
  `offlineMirror` gate, `git fetch` + `git show origin/<base>:ROADMAP.md` read,
  guarded fast-forward of the local base, drift/converged/skip reporting. Reuses
  `scripts/atelier-task-backend` to resolve the backend.
- `scripts/atelier-task-backend` — reused as-is to resolve the backend; may gain
  an `--offline-mirror`/companion read of `.offlineMirror`, or the helper reads
  that key directly with `jq` (no change to the backend script's contract).
- `hooks/tests/mirror-reconcile.test.sh` *(new)* — hermetic test (no network):
  fixture repos for `files`, `github-project + offlineMirror`, and a worktree
  layout; asserts source = `origin/<base>` and the path glob is not relied on.
- `commands/sync-mirror.md` *(new, optional)* — operator-facing on-demand
  reconcile wrapping the helper (housekeeping-style hook↔command split).
- `PLAN.md` §16 — new sub-section documenting the convergent
  reconcile-from-`origin/<base>`-on-`SessionStart` design and its relationship to
  the per-PR write gate (#32) and crt's reverse engine (§16.6).
- **crt boundary (not edited here):** the `backend → files` reverse engine and
  `/migrate-roadmap --to files` live in `claude-roadmap-tools`; atelier-core
  *drives* them — it must not re-implement board reads.

## Acceptance criteria

- [ ] After a task is delivered via the real flow (`worktree → PR → merge to
      base`), the **next session** reconciles the github-project mirror with no
      manual `git pull` and no manual sync invocation — the reconcile is
      triggered automatically on `SessionStart`.
- [ ] The reconcile reads the merged truth from `origin/<base>` (via
      `git fetch` + `git show origin/<base>:ROADMAP.md` / `HISTORY.md` /
      `IN_PROGRESS.md`), independent of local-checkout currency and of worktree
      layout — never the local working copy and never a worktree-relative path.
- [ ] Edits that atelier makes inside worktrees
      (`<repo>-worktrees/<branch>/ROADMAP.md`) are **not silently missed**: the
      reconcile does not depend on a main-checkout path glob
      (`*/<repo>/ROADMAP.md`) to fire.
- [ ] The reconcile is a **no-op** for the `files` backend and for any project
      with `offlineMirror: false` (no fetch, no output, exit 0).
- [ ] The local-base fast-forward is **guarded**: a dirty or diverged main
      worktree causes the step to skip with a surfaced note — never a force
      update, never a write to a dirty checkout.
- [ ] Hermetic regression coverage in the structural suite (`hooks/tests/`)
      asserts both the `origin/<base>` source and the `SessionStart` trigger, plus
      the backend/`offlineMirror` gating and worktree-path independence; runs with
      no network.

## Risks / open questions

- **SessionStart cheapness vs. a network fetch — the central design call.**
  `orient-session.sh` and `daily-housekeeping.sh` are deliberately CHEAP +
  FAIL-OPEN and touch **no** `gh`/remote `git`, precisely to avoid adding latency
  and permission/OAuth prompts to every session start. A reconcile that
  `git fetch`es (and, for the board read, would need the GitHub MCP/OAuth) breaks
  that discipline. The plan resolves this by (a) gating the network half behind a
  **once-per-day stamp**, (b) keeping the bash hook to *git only* (fetch +
  `origin/<base>` read + guarded fast-forward) and (c) deferring the OAuth-bound
  board→mirror regen to the AI layer (driving crt). **Open question for the
  product lead:** is a once-per-day `git fetch` on SessionStart acceptable, or
  should the reconcile run only on an explicit `/atelier:sync-mirror` +
  inside `/next-task` (which already fetches `origin/<base>`)? The latter is
  zero-added-latency but loses the "every session converges automatically"
  property the task asks for.
- **Sequencing vs. #32 (no formal `blocked_by`).** #32 makes the closing PR a
  hard gate so `origin/<base>`'s mirror == the board; #34 is the convergent
  read-side backstop. If #32 lands first, #34's cheap `origin/<base>`
  fast-forward covers the common case and the OAuth board-read is pure
  belt-and-suspenders. If #34 lands first, board changes with no regen PR still
  won't surface from `origin/<base>` alone (they need the crt board read). Worth
  the product lead deciding whether to declare `#34 blocked_by:#32` or ship them
  in either order.
- **Where the bash/AI line falls determines #34's delivered surface.** If the
  product lead wants #34 to also drive the crt `backend → files` regen
  automatically (not just fast-forward from `origin/<base>`), that part cannot
  live in a bash hook — it must be an AI-layer step (command/skill) the hook
  *prompts*, since `listTasks` needs the GitHub MCP. The plan assumes the bash
  hook delivers the git half + a surfaced instruction, not an autonomous board
  read. Confirm that scope boundary.
- **Worktree-launched sessions.** A `SessionStart` whose `CLAUDE_PROJECT_DIR` is
  a task worktree should not reconcile (or fast-forward) the main checkout's
  mirror. The helper must detect a worktree dir and stay a no-op there — the
  reconcile is a main-checkout concern. Needs an explicit guard + test fixture.
- **Estimate is a best-effort `~3h`** (the ROADMAP carries `~TODO`): new hook +
  helper + crt-driving wiring + hermetic test + PLAN update. It stays comfortably
  within the 200-line / 10-file `prSize` budget (tests are exempt), so no
  decomposition is warranted; if the AI-driven board-regen half is pulled in
  scope, revisit.
