## Approach

The github-project backend keeps `ROADMAP.md` / `IN_PROGRESS.md` / `HISTORY.md` /
`roadmap/` as an **offline mirror** of the Project board
(`.roadmap.json`: `backend: github-project`, `offlineMirror: true`,
`githubProject.stateMap` = `{Todo→roadmap, In Progress→inProgress, Done→history}`,
Project #1 / AkaLab-Tech). The board is the single source of truth; the mirror is
a cheap, offline surface (read by `scripts/atelier-orient` via `orient-session.sh`,
by `/atelier:status`, etc.). It is never the claim or backlog source —
`/atelier:next-task` already drives the board directly through crt's
`RoadmapBackend`.

**The bug.** The mirror goes stale after a board item moves (e.g. a task lands
and the item moves to `Done`) and stays stale until something re-syncs it. crt
*does* have the re-sync logic — the "Mirror auto-refresh on activation" procedure
— but it is **gated on skill activation**: it runs only when a prompt causes the
`roadmap-tracking-flow` skill to activate. A session that never touches tracking
(most sessions) never triggers it, so the local orientation surface drifts from
the board for the whole session.

**The corrected, narrowed fix.** Surface a `SessionStart`-time instruction that
tells the AI layer to run crt's **EXISTING** mirror auto-refresh — i.e. activate
`roadmap-tracking-flow` / refresh the offline mirror — for non-`files` +
`offlineMirror:true` repos. We do **not** write a new reconcile engine, we do
**not** call `/migrate-roadmap --to files`, we do **not** touch `.roadmap.json`,
and we do **not** flip the backend. We only make the existing, non-destructive
refresh fire when it otherwise would not.

1. **Engine = crt's EXISTING "Mirror auto-refresh on activation"** (SKILL.md
   lines 533–568). It reads the board via `listTasks("roadmap"|"in_progress"|
   "history")`, regenerates the local mirror files in place, and leaves
   `.roadmap.json` intact (read-only mirror refresh). atelier-core **drives** crt
   (the same prose-instruction delegation pattern `/setup-project` uses for
   `/create-roadmap`); it must **not** re-implement board reads, the Projects
   GraphQL calls, or the `backendId` coherence scheme — those are sovereign in
   crt.

2. **Trigger = `SessionStart`** — a **fourth** hook alongside the existing three
   (`hooks/hooks.json`: `load-operator-rules.sh`, `daily-housekeeping.sh`,
   `orient-session.sh`), following the established **hook → helper split**
   (`orient-session.sh` → `scripts/atelier-orient`; `daily-housekeeping.sh`'s
   daily-stamp discipline). New hook `hooks/refresh-mirror.sh` delegates to a new
   helper `scripts/atelier-refresh-mirror`. The trigger is the session itself, so
   the refresh no longer depends on the prompt happening to mention tracking.

3. **The Bash / AI split (the load-bearing design decision).** crt's refresh
   reads the board over the **GitHub MCP / OAuth** (SKILL.md `isAvailable()`
   requires a registered GitHub MCP; PLAN.md §16.3), which a bash `SessionStart`
   hook **cannot** drive — and the existing cheap hooks deliberately touch no
   `gh` / remote `git` to avoid latency, permission prompts, and premature OAuth.
   So the work is split exactly along that line:
   - **Bash half (this task's hook + helper) — a thin, fail-open surfacer.** It
     does the cheap gating and, when due, **prints a one-line instruction** to
     the session telling the AI layer to run crt's existing mirror auto-refresh.
     It performs **no** `git fetch`, **no** `origin` read, **no** board read, and
     **no** `/migrate-roadmap`. Per the SessionStart contract the helper's stdout
     becomes session context (same mechanism as `orient-session.sh`), so the
     instruction reaches the model.
   - **AI half (board → mirror regen).** Acting on the surfaced instruction, the
     session activates `roadmap-tracking-flow` and runs its existing auto-refresh
     procedure, which reads the board over the MCP and rewrites the local mirror
     **without** removing `.roadmap.json`. This is where the OAuth/MCP-bound work
     lives, by necessity.

   The line is documented: the bash hook owns *gating + a surfaced instruction*;
   the AI layer owns *the board read + the file regen via crt's existing engine*.
   The bash hook is never autonomous over the board.

4. **Gating — non-`files` + `offlineMirror: true` only.** The helper resolves the
   backend via `scripts/atelier-task-backend "$dir"` and reads `.offlineMirror`
   from `.roadmap.json` with `jq`. It is a **silent no-op** (no output, exit 0)
   for the `files` backend and for any project with `offlineMirror: false` /
   absent — no refresh instruction is surfaced for those.

5. **Cheap / fail-open.** A **once-per-day stamp** under `$ATELIER_CONFIG_DIR`
   (the exact `daily-housekeeping.sh` pattern: stamp file,
   `[ "$LAST" = "$TODAY" ] && exit 0`) bounds the per-session cost so the refresh
   instruction is surfaced at most once per calendar day, not on every session
   start. Everything **fails open** (exit 0, silent) on any missing prerequisite
   — no `$ATELIER_CONFIG_DIR`, no `jq`, no `date`, no helper — so a session never
   blocks or errors on the refresh. (If the GitHub MCP is unavailable when the AI
   layer acts, crt's own auto-refresh already falls back gracefully to the
   existing snapshot per SKILL.md lines 547–553 — atelier adds nothing there.)

6. **Worktree guard.** A session whose `CLAUDE_PROJECT_DIR` is a **task worktree**
   (`<repo>-worktrees/<branch>/…`) must **not** refresh the main checkout's mirror
   — the mirror is a main-checkout concern and a worktree is mid-task. The helper
   detects a linked worktree cheaply and offline (`.git` is a file gitdir-pointer
   rather than a directory, equivalently `git rev-parse --git-common-dir` ≠
   `--git-dir`) and stays a no-op there.

Steps, in order:
1. `scripts/atelier-refresh-mirror` (new helper): resolve `$dir`; worktree guard
   → no-op; resolve backend + `.offlineMirror`, no-op unless
   `non-files && offlineMirror`; once-per-day stamp guard; on a stale stamp, print
   the one-line instruction naming crt's EXISTING mirror auto-refresh (explicitly
   NOT `/migrate-roadmap --to files`) and stamp today's date. `--help`,
   `set -uo pipefail`, exit 0 always (fail-open), `$CLAUDE_PLUGIN_ROOT`-relative —
   mirror the `atelier-orient` / `atelier-task-backend` conventions exactly.
2. `hooks/refresh-mirror.sh` (new hook): cheap dispatcher; resolve the helper
   (PATH then `$CLAUDE_PLUGIN_ROOT/scripts`), `[ -d "$ATELIER_CONFIG_DIR" ]` /
   `[ -x ]` fail-soft, print its stdout. Mirror `orient-session.sh` structure
   line-for-line.
3. Register the hook as a fourth `SessionStart` entry in `hooks/hooks.json`.
4. Hermetic regression test in `hooks/tests/` (see Acceptance): backend +
   `offlineMirror` gating, instruction names crt's existing auto-refresh and NOT
   `/migrate-roadmap`, asserts nothing touches/removes `.roadmap.json`, worktree
   no-op, daily-stamp bounded, SessionStart-wired.
5. Document the SessionStart-surfaced refresh under PLAN.md §16 (a short
   sub-section near §16.6), recording the Bash/AI split and that it drives crt's
   **existing** non-destructive auto-refresh — never the authority-flipping
   `/migrate-roadmap --to files` path.

## Affected areas

- `hooks/hooks.json` — add a fourth `SessionStart` hook entry pointing at
  `"${CLAUDE_PLUGIN_ROOT}"/hooks/refresh-mirror.sh`.
- `hooks/refresh-mirror.sh` *(new)* — thin, fail-open `SessionStart` dispatcher
  with no git/gh of its own; delegates to the helper and echoes its surfaced
  instruction. Patterned on `hooks/orient-session.sh`.
- `scripts/atelier-refresh-mirror` *(new)* — the helper: worktree guard, backend
  + `.offlineMirror` gate, once-per-day stamp, and the surfaced one-line "run
  crt's existing mirror auto-refresh" instruction. Reuses
  `scripts/atelier-task-backend` to resolve the backend; reads `.offlineMirror`
  with `jq`. No network, no fetch, no `origin` read, no `/migrate-roadmap`.
- `scripts/atelier-task-backend` — reused as-is to resolve the backend (no change
  to its contract; the helper reads `.offlineMirror` directly with `jq`).
- `hooks/tests/refresh-mirror.test.sh` *(new)* — hermetic test (no network):
  fixtures for `files` backend, `github-project + offlineMirror:true`,
  `github-project + offlineMirror:false`, and a linked-worktree layout; asserts
  the instruction names crt's existing auto-refresh, never names
  `/migrate-roadmap --to files`, and that the helper never reads/removes
  `.roadmap.json`.

- **Optional, product-lead's call:** `commands/sync-mirror.md` *(new)* — a thin
  operator-facing `/atelier:sync-mirror` that, on demand, tells the AI layer to

  `daily-housekeeping.sh` ↔ `/atelier:housekeeping`), for converging the mirror
  mid-day without waiting for the daily stamp. Pure convenience; does not change
  the engine.
- **crt boundary (not edited here):** the mirror auto-refresh engine lives in
  `claude-roadmap-tools` `skills/roadmap-tracking-flow/SKILL.md`; atelier-core
  *drives* it and must not re-implement board reads or the `.roadmap.json`

## Acceptance criteria

- [ ] On `SessionStart` in a non-`files` repo with `offlineMirror: true`, a
      one-line instruction is surfaced telling the session to run crt's mirror
      auto-refresh, so the mirror converges **without** the prompt having to
      mention tracking (closing the activation-gated gap) — at most once per
      calendar day.
- [ ] The refresh drives crt's **EXISTING** "Mirror auto-refresh on activation"
      procedure (SKILL.md §"Mirror auto-refresh on activation", `listTasks` across
      the `roadmap` / `in_progress` / `history` buckets) — **not** a new
      reconcile engine written in atelier-core.
- [ ] The fix **NEVER** uses `/migrate-roadmap --to files` (crt step 5d) — neither
      the bash helper nor the surfaced instruction references it; the hermetic
      test asserts that string never appears.
- [ ] The fix **NEVER** removes, mutates, or rewrites `.roadmap.json`, and
      **NEVER** flips the backend away from `github-project` — the github-project
      backend stays the source of truth; only the read-only mirror files are
      regenerated.
- [ ] atelier-core **drives** crt and does not re-implement the board read, the
      Projects GraphQL calls, or the `backendId` coherence scheme.
- [ ] The refresh is a **no-op** (no output, exit 0) for the `files` backend and
      for any project with `offlineMirror: false` / absent.
- [ ] **Worktree guard:** a `SessionStart` whose `CLAUDE_PROJECT_DIR` is a task
      worktree does not surface a refresh for the main checkout's mirror — it is a
      no-op there.
- [ ] **Cheap / fail-open:** a once-per-day stamp bounds the per-session cost; on
      missing prerequisites (no `$ATELIER_CONFIG_DIR`, no `jq`, no `date`, no
      helper) and on an unavailable GitHub MCP the path exits 0 silently and never
      blocks the session (crt's own auto-refresh handles the MCP-unavailable
      fallback to the existing snapshot).
- [ ] **Bash/AI split is explicit:** the bash hook surfaces the one-line
      instruction and the AI layer performs the board read + mirror regen via
      crt's existing engine; the bash hook never drives the GitHub MCP/OAuth
      itself. Documented in PLAN.md §16 + the helper header comment.
- [ ] Hermetic regression coverage in `hooks/tests/` asserts: the SessionStart
      trigger, the backend + `offlineMirror` gating, the worktree no-op, the
      daily-stamp bound, that the instruction names crt's existing auto-refresh,
      and that nothing references `/migrate-roadmap --to files` or touches
      `.roadmap.json`; runs with no network.

## Risks / open questions

- **AI-driven by necessity — confirm the surface.** Because the board read needs
  the GitHub MCP/OAuth, the autonomous regen cannot live in a bash hook; the hook
  can only *prompt* the AI layer to run crt's existing auto-refresh. Reliability
  therefore depends on the model acting on the surfaced SessionStart instruction.
  *Open question:* is "bash surfaces an instruction, AI runs crt's existing
  refresh" the intended delivered surface, or does the product lead want a
  stronger guarantee (e.g. wiring the refresh into `/next-task`, which already
  authenticates to the board for `listTasks`, so the mirror converges as a side
  effect of claiming work)? The latter is zero-extra-OAuth but only converges when
  the operator starts a task, not on every session.
- **Is the hook even needed, or is documentation enough?** Part of the fix may be
  simply strengthening crt/atelier docs so that `roadmap-tracking-flow`'s
  auto-refresh is understood to run on SessionStart for offlineMirror repos. If
  the skill reliably activates early in most real sessions, the marginal value of
  a dedicated hook is the *guaranteed* fire on tracking-silent sessions. *Confirm*
  the hook is worth the fourth SessionStart entry vs. a docs-only nudge.
- **Daily-stamp cadence vs. multi-move days.** A once-per-day stamp may leave the
  mirror stale after a second/third board move in the same day. The optional
  `/atelier:sync-mirror` command covers the on-demand case. *Confirm* the daily
  cadence is the right default, or whether the stamp should be finer-grained.
- **Worktree detection robustness.** The chosen check (`.git` is a file /
  `--git-common-dir` ≠ `--git-dir`) is cheap and offline; it must also no-op
  gracefully outside a git repo. Covered by a test fixture.
- **Mirror-file writes vs. git tracking.** crt's auto-refresh regenerates the
  mirror files in place. On this repo the mirror files are tracked (atelier-dev
  uses the indexed §5 layout); the regen produces ordinary working-tree edits and
  commits nothing on its own. Verify the `Edit/Write` / commit guardrail hooks
  (`scan-edit-write.sh`, `safe-commit.sh`) do not block the regen and that nothing
  auto-commits the refreshed mirror.
- **Estimate `~2.5h`** — smaller than the 2nd plan because there is no new engine
  to build: one new hook + one new helper + one `hooks.json` entry + a hermetic
  test (exempt) + a short PLAN.md sub-section (+ an optional thin command). Well
  under the 200-line / 10-file `prSize` budget — no decomposition warranted.
