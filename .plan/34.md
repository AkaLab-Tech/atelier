# Plan — #34 github-project offline mirror must reconcile from the BOARD on `SessionStart` (crt reverse engine), not from `origin/<base>`

- **Status:** ready (approved — product lead)
- **Type / priority / estimate:** bug · P1 · ~3h
- **blocked_by:** none — but see *Risks* (depends on crt's `github-project → files` reverse path; companion `#32` is now obsolete)

> **Re-plan note (premise change).** This OVERWRITES the previously-approved
> `#34` plan. That plan reconciled the mirror from `origin/<base>` —
> `git fetch` + `git show origin/<base>:ROADMAP.md` + a guarded fast-forward of
> the local base. A **new operator policy (PR #272)** makes the roadmap files
> mirror (`ROADMAP.md`, `IN_PROGRESS.md`, `HISTORY.md`, `roadmap/`)
> **LOCAL-ONLY**: it is removed from GitHub (`git rm --cached` + ignored via
> `.git/info/exclude`, *not* `.gitignore`) so the GitHub Project board is the
> single source of truth. There is therefore **no mirror on `origin` to read or
> fast-forward from** — the old central mechanism is structurally impossible.
> The companion task `#32` (closing-PR regenerates the mirror) was closed
> obsolete for the same reason. This plan re-sources the reconcile from the
> **board**, driven through crt's reverse engine.

## Approach

The github-project backend keeps `ROADMAP.md` / `IN_PROGRESS.md` / `HISTORY.md` /
`roadmap/` as an **offline mirror** of the Project board
(`.roadmap.json`: `backend: github-project`, `offlineMirror: true`,
`githubProject.stateMap` = `{Todo→roadmap, In Progress→inProgress, Done→history}`).
The board — Project #1, AkaLab-Tech — is the single source of truth. The mirror is
a convenience surface read by cheap, offline tooling (`scripts/atelier-orient`
via the `orient-session.sh` hook, `/atelier:status`); it is never the claim or
backlog source. `/atelier:next-task` already drives the board directly for this
backend (`commands/next-task.md` lines 47/69: `listTasks("roadmap")` through
`claude-roadmap-tools`' `RoadmapBackend`, *not* `origin/<base>:ROADMAP.md`), so
the mirror's only job is to keep the local orientation surface honest.

**The bug.** The local mirror goes stale after a task is delivered
(`worktree → PR → merge to base`, then the board item moves to `Done`) and stays
stale until someone manually re-syncs. Two structural causes (TASK_034, deminut
`#28` evidence): the prior reconcile read the **local working checkout** (which
lags the merged truth) and a change-detection hook was keyed to a
**main-checkout path glob** (`*/<repo>/ROADMAP.md`) that never matches atelier's
own edits because they happen inside worktrees
(`<repo>-worktrees/<branch>/ROADMAP.md`).

**The corrected fix.** Reconcile the mirror **from the board**, on `SessionStart`,
driving crt's reverse engine — never from `origin` and never from the local
checkout:

1. **Source = the GitHub Project BOARD (truth), via crt's reverse engine.**
   Regenerate the local mirror with the `backend → files` reverse path
   (`RoadmapBackend.listTasks` across the three buckets — `Todo`/`Backlog`,
   `In Progress`, `Done`), driven through crt's `/migrate-roadmap --to files`.
   atelier-core **drives** crt (the same prose-instruction delegation pattern
   `/setup-project` uses for `/create-roadmap` and `/onboard-workspace` uses for
   `/adopt-roadmap`); it must **not** re-implement board reads, the GraphQL
   Projects calls, or the `backendId` coherence scheme — those are sovereign in
   `claude-roadmap-tools` (PLAN.md §16.3 / §16.6). The reverse path
   regenerates the local mirror files in place (they are git-excluded local-only
   artifacts, so the regen writes untracked working-tree files and commits
   nothing).

2. **Trigger = `SessionStart`** — a **fourth** hook alongside the existing three
   (`hooks/hooks.json`: `load-operator-rules.sh`, `daily-housekeeping.sh`,
   `orient-session.sh`), following the established **hook → helper split**
   (`orient-session.sh` → `scripts/atelier-orient`; `daily-housekeeping.sh` →
   `scripts/atelier-housekeeping`). New hook `hooks/reconcile-mirror.sh`
   delegates to a new helper `scripts/atelier-reconcile-mirror`. No path glob is
   involved — the trigger is the session itself, so worktree-located edits can
   never be silently missed.

3. **The Bash / AI split (the load-bearing design decision).** The board read
   needs the **GitHub MCP / OAuth** (PLAN.md §16.3: auth via a GitHub MCP, not
   `gh`), which a bash `SessionStart` hook **cannot** drive — and the existing
   cheap hooks deliberately touch no `gh`/remote `git` to avoid latency,
   permission prompts, and premature OAuth. So the reconcile is split exactly
   along that line:
   - **Bash half (this task's hook + helper) — a thin, fail-open dispatcher.**
     It does the cheap gating and, when due, **surfaces a one-line instruction**
     to the session telling the AI layer to run the crt reverse engine. It does
     **no** `git fetch`, reads **no** `origin/<base>` mirror (none exists), and
     performs **no** fast-forward. Per the SessionStart contract, the helper's
     stdout becomes session context (same mechanism as `orient-session.sh`), so
     the surfaced instruction reaches the model.
   - **AI half (board → mirror regen).** Acting on the surfaced instruction, the
     session runs crt's `/migrate-roadmap --to files`, which reads the board
     over the MCP and rewrites the local mirror. This is where the
     OAuth/MCP-bound work lives, by necessity.

   **Where the line falls, documented:** the bash hook owns *gating + a surfaced
   instruction*; the AI layer owns *the board read + the file regen via crt*. The
   bash hook is never autonomous over the board.

4. **Gating — non-`files` + `offlineMirror: true` only.** The helper resolves the
   backend via `scripts/atelier-task-backend "$dir"` and reads `.offlineMirror`
   from `.roadmap.json` with `jq`. It is a **silent no-op** (no output, exit 0)
   for the `files` backend and for any project with `offlineMirror: false` — no
   board read is ever surfaced for those.

5. **Cheap / fail-open.** A **once-per-day stamp** under `$ATELIER_CONFIG_DIR`
   (the exact `daily-housekeeping.sh` pattern: stamp file,
   `[ "$LAST" = "$TODAY" ] && exit 0`) bounds the per-session cost so the
   board-read instruction is surfaced at most once per calendar day, not on every
   session start. Everything **fails open** (exit 0, silent) when prerequisites
   are missing — no `$ATELIER_CONFIG_DIR`, no `jq`, no `date`, no MCP/auth — so a
   session never blocks or errors on the reconcile.

6. **Worktree guard.** A session whose `CLAUDE_PROJECT_DIR` is a **task worktree**
   (`<repo>-worktrees/<branch>/…`) must **not** reconcile the main checkout's
   mirror — the mirror is a main-checkout concern. The helper detects a linked
   worktree cheaply and offline (`git rev-parse --git-common-dir` differs from
   `--git-dir`, equivalently `.git` is a *file* gitdir-pointer rather than a
   directory) and stays a no-op there.

Steps, in order:
1. `scripts/atelier-reconcile-mirror` (new helper): resolve `$dir`; worktree
   guard → no-op; resolve backend + `.offlineMirror`, no-op unless
   `non-files && offlineMirror`; once-per-day stamp guard; on a stale stamp,
   print the one-line crt-reverse-engine instruction and stamp today's date.
   `--help`, `set -uo pipefail`, exit 0 always (fail-open),
   `$CLAUDE_PLUGIN_ROOT`-relative — mirror the `atelier-orient` /
   `atelier-task-backend` conventions exactly.
2. `hooks/reconcile-mirror.sh` (new hook): cheap dispatcher; resolve the helper
   (PATH then `$CLAUDE_PLUGIN_ROOT/scripts`), `[ -d "$ATELIER_CONFIG_DIR" ]` /
   `[ -x ]` fail-soft, print its stdout. Mirror `orient-session.sh`
   structure line-for-line.
3. Register the hook as a fourth `SessionStart` entry in `hooks/hooks.json`.
4. Hermetic regression test in `hooks/tests/` (see Acceptance) — backend +
   `offlineMirror` gated, board-sourced (asserts NO `origin` read), worktree
   no-op, daily-stamp bounded, SessionStart-wired.
5. Document the board-sourced reconcile under PLAN.md §16 (a sub-section near
   §16.6, the reverse-path home), explicitly recording that the local-only
   mirror policy supersedes any `origin/<base>` reconcile and that `#32` is
   obsolete — CLAUDE.md requires the plan and the design note land together.

## Affected areas

- `hooks/hooks.json` — add a fourth `SessionStart` hook entry pointing at
  `"${CLAUDE_PLUGIN_ROOT}"/hooks/reconcile-mirror.sh`.
- `hooks/reconcile-mirror.sh` *(new)* — thin, fail-open `SessionStart` dispatcher
  with no git/gh of its own; delegates to the helper and echoes its surfaced
  instruction. Patterned on `hooks/orient-session.sh`.
- `scripts/atelier-reconcile-mirror` *(new)* — the reconcile helper: worktree
  guard, backend + `.offlineMirror` gate, once-per-day stamp, and the surfaced
  one-line "run the crt reverse engine" instruction. Reuses
  `scripts/atelier-task-backend` to resolve the backend; reads `.offlineMirror`
  with `jq`. No network, no fetch, no `origin` read.
- `scripts/atelier-task-backend` — reused as-is to resolve the backend (no change
  to its contract; the helper reads `.offlineMirror` directly with `jq`).
- `hooks/tests/reconcile-mirror.test.sh` *(new)* — hermetic test (no network):
  fixtures for `files` backend, `github-project + offlineMirror:true`,
  `github-project + offlineMirror:false`, and a linked-worktree layout; asserts
  the instruction names the board / crt reverse engine and that nothing reads
  `origin`.
- `commands/sync-mirror.md` *(new, optional)* — operator-facing on-demand
  reconcile that drives crt's `/migrate-roadmap --to files` directly (the
  hook ↔ command split mirroring `daily-housekeeping.sh` ↔ `/atelier:housekeeping`),
  for the operator who wants to converge the mirror mid-day without waiting for
  the daily stamp.
- `PLAN.md` §16 — new sub-section documenting the board-sourced,
  `SessionStart`-triggered reconcile, the Bash/AI split, and that the local-only
  mirror policy supersedes the `origin/<base>` reconcile premise (and obsoletes
  `#32`).
- **crt boundary (not edited here):** the `backend → files` reverse engine and
  `/migrate-roadmap --to files` live in `claude-roadmap-tools`; atelier-core
  *drives* them — it must not re-implement board reads or the `.roadmap.json`
  contract.

## Acceptance criteria

- [ ] After a task is delivered via the real flow (`worktree → PR → merge to
      base`, board item moved to `Done`), the **next session** reconciles the
      github-project mirror with no manual re-sync — the reconcile is triggered
      automatically on `SessionStart`.
- [ ] The reconcile's source is the **GitHub Project board** (crt's
      `backend → files` reverse engine / `listTasks` across the `Todo`,
      `In Progress`, `Done` buckets), driven through `claude-roadmap-tools` —
      **never** `origin/<base>` (no mirror exists there under the local-only
      policy) and **never** the local working checkout.
- [ ] atelier-core **drives** crt and does not re-implement the board read, the
      Projects GraphQL calls, or the `backendId` coherence scheme.
- [ ] The reconcile does **not** depend on a main-checkout path glob
      (`*/<repo>/ROADMAP.md`) to fire — worktree-located edits are never silently
      missed because the trigger is the session itself.
- [ ] The reconcile is a **no-op** (no output, exit 0) for the `files` backend
      and for any project with `offlineMirror: false`.
- [ ] **Worktree guard:** a `SessionStart` whose `CLAUDE_PROJECT_DIR` is a task
      worktree does not reconcile the main checkout's mirror — it is a no-op
      there.
- [ ] **Cheap / fail-open:** a once-per-day stamp bounds the per-session cost
      (the board-read instruction is surfaced at most once per calendar day); on
      no network / no auth / missing prerequisites the reconcile exits 0
      silently and never blocks the session.
- [ ] **Bash/AI split is explicit:** the bash hook surfaces the one-line
      instruction and the AI layer performs the board read + mirror regen via
      crt; the bash hook never drives the GitHub MCP/OAuth itself. The split is
      documented (PLAN.md §16 sub-section + helper header comment).
- [ ] Hermetic regression coverage in `hooks/tests/` asserts the board source
      (and that nothing reads `origin`), the `SessionStart` trigger, the backend
      + `offlineMirror` gating, and the worktree no-op; runs with no network.

## Risks / open questions

- **crt reverse-path availability (the real dependency).** This plan assumes
  crt's `github-project → files` reverse engine (`/migrate-roadmap --to files`,
  PLAN.md §16.6, marked 🟡 Phase 9.4) is shipped and reliable. If it is not yet
  available for `github-project`, `#34` is effectively **blocked on that crt
  capability** — atelier-core must not work around it by reading the board
  itself. *Product-lead call:* confirm crt provides `github-project → files`
  today, or declare a `blocked_by` on the crt reverse-path task.
- **The reconcile is now AI-driven by necessity — confirm the surface.** Because
  the board read needs the MCP/OAuth, the autonomous regen cannot live in a bash
  hook; the hook can only *prompt* the AI layer. Reliability therefore depends on
  the model acting on the surfaced SessionStart instruction. *Open question:* is
  "bash surfaces an instruction, AI runs crt" the intended delivered surface, or
  does the product lead want a stronger guarantee (e.g. wiring the regen into
  `/next-task`, which already authenticates to the board for `listTasks`, so the
  mirror converges as a side effect of claiming work)? The latter is
  zero-extra-OAuth but only converges when the operator starts a task, not on
  every session.
- **`#32` is obsolete, so there is no per-PR write backstop.** Under the old
  design `#32` (closing-PR regenerates the mirror) was the write-side gate and
  `#34` the read-side backstop. The local-only policy closed `#32`, so this
  SessionStart reconcile is now the **sole** mirror-convergence mechanism. Worth
  the product lead noting that between deliveries the local mirror is expected to
  be stale until the next daily reconcile — acceptable because the board, not the
  mirror, is the source of truth.
- **Daily-stamp cadence vs. multi-delivery days.** A once-per-day stamp may leave
  the mirror stale after a second/third delivery in the same day. The optional
  `/atelier:sync-mirror` command (or invoking crt's `/migrate-roadmap --to files`
  directly) covers the on-demand case. *Confirm* the daily cadence is the right
  default, or whether the stamp should be finer-grained.
- **Excluded-file writes must not trip guardrails.** The mirror files are
  git-excluded local-only artifacts (`.git/info/exclude`). The crt regen writes
  them as untracked working-tree files and commits nothing; verify the
  `Edit/Write`/commit guardrail hooks (`scan-edit-write.sh`, `safe-commit.sh`)
  do not block writing excluded mirror files and that nothing attempts to commit
  them.
- **Worktree detection robustness.** The chosen check
  (`git rev-parse --git-common-dir` ≠ `--git-dir`, i.e. `.git` is a file) is
  cheap and offline; it must also no-op gracefully outside a git repo. Covered by
  a test fixture.
- **Estimate `~3h`** (the board carries `~TODO`): new hook + helper + one
  `hooks.json` entry + a hermetic test (exempt) + a PLAN.md sub-section. Well
  under the 200-line / 10-file `prSize` budget — no decomposition warranted.
