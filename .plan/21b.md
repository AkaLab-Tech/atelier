## Risks / open questions

- **Round-trip exactness vs slug regeneration.** `TASK_NNN` is preserved, but `<slug>` is derived from the title — if a title changed remotely, the reconstructed filename slug may differ from the original. Confirm "lossless" is keyed on `TASK_NNN` + content, not on the exact filename slug (recommend: yes, slug is cosmetic).
- **`linear ↔ github-project` via files is two manual commands, not one.** The plan documents the route but does **not** automate it. Confirm that's acceptable for v1 (the briefing leans this way — "likely still ❌, route via files").
- **Status round-trip when `stateMap` has multiple values per bucket.** Forward push uses `stateMap[bucket][0]`; reverse-lookup maps *any* state in `stateMap[bucket]` back to the bucket. A task that was in `Todo` (also a `roadmap` state) round-trips to the `roadmap` section but loses the `Todo`-vs-`Backlog` distinction (both are `roadmap`). This is inherent to the bucket model — document it under "inherently lossy."
