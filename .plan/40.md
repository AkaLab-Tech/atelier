## Approach (reconstructed from the title)

Establish a mandatory CI/CD standard that every atelier-managed project's pipeline enforces the 8 named dimensions — **checks, lint, typecheck, tests, validate, security, size, review** — as a first-class, default part of the project lifecycle (mirroring how `#45`'s branch-protection task makes branch protection first-class rather than opt-in). Reading each dimension against what atelier already partially has:

1. **checks** — CI status checks exist (`.github/workflows/structural.yml` for atelier itself); per-project equivalent is `/atelier:validate`'s lint/typecheck/test bundle, already wired into the push gate (PLAN.md §6).
2. **lint** — per-project lint, already required by the push gate.
3. **typecheck** — same, already required.
4. **tests** — unit/integration, already required by the push gate; e2e (Playwright) required by the PR gate.
5. **validate** — `/atelier:validate` as the umbrella command; the epic likely means "every project's CI actually runs `/atelier:validate` (or its per-project equivalent) as a gate, not just locally."
6. **security** — no first-class per-project security scanning exists today (atelier itself has no `security` CI step beyond the structural workflow's checks) — this is the most likely **net-new** dimension: dependency vuln scanning (mirroring `/safe-install`'s guardrails, but as a recurring CI check, not just at install time), secret scanning beyond the existing `block-env-commit`/`scan-git-add` hooks.
7. **size** — `atelier-pr-size-check` already exists per-project; this dimension is likely "confirm it's wired into every project's CI, not just the local hook."
8. **review** — the existing `reviewer` agent + auto-merge guardrail #2; overlaps with `#38`'s self-approval problem — this epic likely assumes `#38`/`#44` land first so "review" is a working gate everywhere, not a separate build.

Given 6 of 8 dimensions (checks/lint/typecheck/tests/validate/size) already exist per-project via the push/PR gates and `/atelier:validate`, the **actual net-new work is likely narrower than the epic title suggests**: (a) a per-project CI workflow template that runs `/atelier:validate` (not just relies on local hooks) so gates are enforced even if an agent bypasses local hooks, (b) a security-scanning step, and (c) confirming `#45` (branch protection) and `#38`/`#44` (review) are prerequisites rather than sub-scope here.

## Affected areas (reconstructed — confirm before build)

- `templates/` — a new per-project CI workflow template (e.g. `templates/ci-workflow.yml`) that `/atelier:setup-project` instantiates, running lint/typecheck/test/validate/size/security as GitHub Actions checks.
- `scripts/atelier-setup-project` — instantiate the new CI template by default.
- `scripts/atelier-doctor` — detect a project missing the standard CI workflow, `--fix` scaffolds it (mirroring the `#45` branch-protection pattern).
- `commands/align.md` — propagate the CI standard across already-registered projects (again mirroring `#45`).
- New security-scanning mechanism (dependency + secret) — likely a new script or a step wired into the new CI template; needs its own tool choice via `/safe-install`.
- PLAN.md — record the standard as a decided contract.
