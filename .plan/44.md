## The problem being fixed (and the wrong fix already shipped)

atelier enforces two-party review on two orthogonal axes: **git identity** (GitHub `reviewDecision` — satisfied by the dual `gh/author` + `gh/reviewer` accounts) and **actor/session** (Claude Code's auto-mode classifier). The classifier is **session+intent-scoped**: it attributes a sub-agent's `git push` / `gh pr create` back to the *orchestrating session* and reads intent across the whole conversation. So a driving/main session that dispatches an authoring sub-agent and then turns around and dispatches `reviewer`/`auto-merge` on that same PR is **still** flagged as self-approving. Delegating only the authoring is insufficient — this was disproven empirically by PR #293's own blocked reviewer dispatch.

**PR #293's authoring-only conclusion is already merged into `main`** and is wrong: `agents/pr-opener.md` exists, `commands/align.md` Tier-3 `auto` delegates authoring to `pr-opener` and then dispatches `reviewer`/`auto-merge` **from the align session itself**, `operator-rules.md` § "PR authoring is always sub-agent work" + `PLAN.md §7` encode the incomplete invariant, and `hooks/tests/pr-opener-delegation.test.sh` locks it in. `#44` corrects all of this.

## Chosen design (FIXED by operator — Option B)

Generalize `task-orchestrator` with a **non-task PR coordination mode**: it coordinates author→review→auto-merge for a non-task branch (`chore/*`, a plan-tracking branch, an ad-hoc `fix/*`) with NO per-task worktree creation, NO `IN_PROGRESS → HISTORY` move, and NO `.plan` lookup. The generalization is **strictly additive and guarded** so the `/next-task` path cannot regress. `pr-opener` is kept as the authoring primitive the generalized orchestrator dispatches for non-task branches (`pr-author` stays the `task/*` specialist). The standalone flows (`/atelier:align` Tier-3 `auto`, the `/plan-task` tracking PR, `/atelier:release`, ad-hoc requests) then delegate the **whole** coordination to the orchestrator, exactly as `/next-task` does — so no driving session ever coordinates both author and review of the same PR.

Option A (a separate `pr-orchestrator` agent) was rejected; do not re-open it.
