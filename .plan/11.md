# Plan — #11 Iterate the network allowlist

- **Status:** ready (approved — product lead, batch approval via AskUserQuestion, 2026-07-07)
- **Type / priority / estimate:** chore · P2 · ~1h (this pass; the task is inherently ongoing)
- **blocked_by:** none

## Approach

This is a standing chore ("grow the allowlist organically based on what M7.1 needs"), not a one-shot deliverable — each iteration adds justified entries as new legitimate network needs surface. This plan covers the **first concrete pass**: audit `templates/settings.template.json`'s current network allowlist against what atelier's shipped integrations (GitHub, Coolify, Vercel, Neon, Linear MCP) actually reach, and either (a) confirm it's already sufficient and close with a "no gaps found" note, or (b) add the missing host(s) with a one-line justification comment next to each new entry, per the task's own acceptance bar.

Since this chore has no natural "done" state, close *this* task once the audit pass lands, and leave the underlying need (grow organically as new integrations are added) as implicit, ongoing maintenance rather than a task that stays open forever — a future integration that needs a new host gets its own follow-up entry when it happens.

## Affected areas

- `templates/settings.template.json` — network allowlist section, only if the audit finds a real gap.
- A short note in `PLAN.md` or `docs/troubleshooting.md` recording the audit result and the "add a one-line justification per entry" convention, so future additions follow the same discipline.

## Acceptance criteria

- [ ] Every currently-shipped integration (GitHub/gh, Coolify, Vercel, Neon, Linear MCP) is checked against the allowlist; any gap is closed with a justified addition.
- [ ] Every new entry (if any) carries a one-line justification.
- [ ] A short note records the convention for future additions (so this doesn't need re-litigating each time).

## Risks / open questions

- **Scope creep risk.** "Grow organically" has no natural stopping point; this plan deliberately scopes to one audit pass against today's shipped integrations, not a speculative future-proofing exercise.
