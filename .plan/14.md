## Approach

Track per-task token/cost usage and enforce an optional budget cap, mirroring the fixed 3+3 retry budget (PLAN.md §8):

1. **Accumulate usage.** Each specialist dispatch (`implementer`/`tester`/`pr-author`/`reviewer`/etc.) already produces a `.task-log/<timestamp>-<NN>.md` entry (per the retry-budget mechanism). Extend that write path (or add a sibling `.task-log/cost.jsonl`) to record token/cost per dispatch — sourced from whatever the `Task` tool / Agent SDK already exposes per-call, not a new estimation mechanism.
2. **Optional cap.** New `.atelier.json` block `taskBudget: { maxTokens, maxUSD }`. Absent → today's behavior, unchanged.
3. **Enforcement point.** `task-orchestrator` checks cumulative spend before each specialist dispatch; on exceeding the cap, stop the chain and write a budget-exceeded log entry (same shape/location as the hard-stop `blocked` path, but a distinct reason string so it's not confused with a real failure) and hand back to the operator rather than to `unblocker` (this is a budget policy stop, not a competence failure).
4. **Reporting.** Per-task cost is recorded in `.task-log/` regardless of whether a cap is configured, so the operator can observe real spend before setting a cap.

## Affected areas

- `agents/task-orchestrator.md` — cumulative-spend check before each dispatch; budget-exceeded terminal state.
- `.atelier.json` schema / `templates/` — new optional `taskBudget` block.
- `.task-log/` write path (wherever attempt logs are currently written) — add token/cost fields.
- `PLAN.md` §8 — document the budget-exceeded terminal state alongside the existing hard-stop path.
- New hermetic test verifying: no cap configured → unchanged behavior; cap configured + exceeded → chain stops with a budget-exceeded log entry, not a `blocked` issue.

## Acceptance criteria

- [ ] A task that exceeds its configured budget halts with a budget-exceeded log entry (distinct from the hard-stop/`blocked` path).
- [ ] With no budget configured, the chain behaves exactly as today.
- [ ] Per-task cost is recorded in `.task-log/` regardless of whether a cap is set.
