# Plan — #39 repo-less install: bootstrap.sh + marketplace-cache runtime, drop the manual clone

- **Status:** ready (operator-approved 2026-07-02; full plan also resident in the board item body)
- **Type / priority / estimate:** feat · P1 · ~3 PRs' worth (delivered as one `task/39-*` PR with phase-per-commit structure)
- **blocked_by:** none. (#37 backend-resident plans is related but not blocking — this file satisfies the current §16.5 plan-on-base gate.)

## Goal

Install atelier without the operator ever running `git clone`. One documented line replaces the clone+`./install.sh` flow.

## Problem

The manual clone at `~/atelier` plays four roles today: (1) delivers `install.sh`; (2) is the *runtime* of the 26 `scripts/atelier-*` helpers — `~/.local/bin` symlinks resolve back into it (`_phase_c_1_symlink_helper`); (3) is the update channel (`atelier-update` does fetch + `merge --ff-only` on it); (4) is the templates source. Meanwhile Phase C.2 already installs the plugin layer from the `akalab-tech` marketplace — and since the plugin IS the whole repo, Claude Code's plugin cache already contains `scripts/`, `templates/` and `install.sh` too. The clone duplicates content the cache ships, and `atelier-update` carries clone↔cache drift-reconciliation complexity that exists only because there are two copies.

## Approach — plugin cache as the single source

```
curl -fsSL <stable-url>/bootstrap.sh | bash
  └─ bootstrap.sh (~100 lines, auditable)
       ├─ minimal deps + Claude Code
       ├─ claude plugin marketplace add AkaLab-Tech/claude-plugins
       ├─ claude plugin install atelier@akalab-tech        → plugin cache (full repo)
       └─ exec <cache>/install.sh --from-cache             → full phased installer as today
             └─ COPIES scripts/ + templates/ →  ~/.local/share/atelier/<version>/
                 current → <version>   (atomic symlink swap; previous version kept for rollback)
                 ~/.local/bin/atelier-* → current/scripts/*
```

Key design decision: helpers are **copied** out of the plugin cache into a versioned runtime dir, never symlinked into the cache — the cache layout is a Claude Code implementation detail; the copy decouples us, and `<version>/` + `current` gives atomic swap and trivial rollback.

## Phases

- **F1 — parameterize the source root.** `install.sh` and `atelier-update` today assume "I live inside a git clone". Introduce a single `ATELIER_SOURCE_ROOT` resolution: git clone (legacy/dev), plugin cache, or unpacked tarball. Prerequisite for everything; no behavior change.
- **F2 — versioned runtime dir.** New installer phase: copy `scripts/` + `templates/` from source → `~/.local/share/atelier/<version>/`, swap `current`, repoint `~/.local/bin` symlinks. Transparent migration: symlinks pointing at a clone get rewritten.
- **F3 — new `bootstrap.sh`.** Tiny, no phase logic: minimal deps (git, jq, Claude Code), marketplace add, plugin install, delegate to the cached `install.sh`. The only curl|bash artifact — keep it small and auditable; document a two-step download→inspect→run path. Consistent with the existing vendor-installer exceptions (Claude Code, fnm).
- **F4 — `atelier-update` without git.** Managed mode: `claude plugin update` → compare cache version vs `current` → copy to `<new-version>/` + swap. Drop clone↔cache drift reconciliation. Keep `--plugin-root` as an explicit legacy/dev mode for one grace release.
- **F5 — doctor + uninstall.** Doctor: detect install mode (clone/managed), verify cache ↔ `current` ↔ symlink coherence, suggest migration to clone-mode installs. Uninstall: clean the versioned runtime dir.
- **F6 — docs.** operator-guide (the `git clone` at the top), README ("nuclear reset"), troubleshooting, CLAUDE.md. Jr install becomes one line.
- **F7 (optional hardening, not in this PR) — verifiable integrity.** Pin the documented bootstrap URL to a tag/SHA; runtime-tree checksum verified by doctor.

## Risks / open decisions

- Plugin-cache layout stability is the main risk — mitigated by the copy (F2). Plan B (cheap thanks to F1): GitHub Releases tarball + CI-generated SHA256SUMS as the source.
- Bootstrap chicken-and-egg (needs `claude` before plugin install) — already solved by Phase 0; extract that slice into bootstrap.sh.
- git-wt is still installed by cloning its repo (SHA-pinned since #286) — same treatment later, out of scope.

## Acceptance

- A machine with no atelier clone installs end-to-end from the documented one-liner; all `atelier-*` helpers work and resolve into `~/.local/share/atelier/current/`.
- `atelier-update` upgrades cache→runtime atomically with rollback, no git operations on any operator-owned checkout.
- `atelier-doctor` reports install mode and cache/runtime/symlink coherence; clone-based installs get a migration hint.
- Docs contain no `git clone` step for operators; hermetic tests cover the cache-sourced install path.
